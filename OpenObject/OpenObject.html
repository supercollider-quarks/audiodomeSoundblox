OpenObject talk to objects over the network


Inherits from: Object


This class allows to publish any object for remote access via network (OSC)


See also: OSC, NetAddr



*start(addr)

Start listening to addr (if nil, it listens to all addresses)


OpenObject.start;


*end

Stop listening.


OpenObject.end;

*put(name, object)

Add an object for remote access under a name

a = [1, 2, 3];

OpenObject.add(\foo, a);

// alternative:

a.publish(\foo);

*remove(object)

Remove an object

OpenObject.remove(a);

// alternative:

a.unpublish;



OSC API


Once an object has been published under a name, methods can be called via OSC messages.


selector: the message name

oo calls arguments in sequence

ook calls keyword arguments, in pairs of key and value.

Array arguments are enclosed in bracket symbols (see example below).


"/oo" name selector arg1 arg2 ...

"/ook" name selector argname1 arg1 argname2 arg2 ...




// example


OpenObject.start;

n = NetAddr("127.0.0.1", NetAddr.langPort); // loopback IP for this test



a = [1, 2, 3, 3, 4];

a.publish(\paris);


n.sendMsg("/oo", \paris, \put, 2, 1974);

a.postln;

n.sendMsg("/oo", \paris, \put, 2, '[', 1974, 1975, 1976, ']');

a.postln;

n.sendMsg("/oo", \paris, \put, '[', 2, 3, ']', 500);

a.postln;



// sound example


Ndef(\y, { |freq=440, numharm = 200| Blip.ar(freq * [1, 1.2], numharm) * 0.3 } );

Ndef(\y).publish(\brussels);

// play and stop

n.sendMsg("/oo", \brussels, \play);

n.sendMsg("/oo", \brussels, \stop);

// using keyword args (/ook)

n.sendMsg("/ook", \brussels, \play, \fadeTime, 3);

// setting args

n.sendMsg("/oo", \brussels, \set, \freq, 210, \numharm, 90);

n.sendMsg("/oo", \brussels, \stop);



// talk to the current environment

currentEnvironment.publish(\envir);

n.sendMsg("/oo", \envir, \put, \x, 210);

currentEnvironment.postcs;

currentEnvironment.unpublish;


Remote controlling proxies 


see also: JITLib


findProxies

findProxies_(boolean)

if set to true, the classes Pdef, Pdefn, Tdef, Ndef and Fdef can be remote controlled 

without explict publishing

// example:

OpenObject.start;

n = NetAddr("127.0.0.1", NetAddr.langPort); // loopback IP for this test


Pdef(\x, Pbind(\dur, Pseries(0.2, 0.1, 6), \note, Prand([0, 2, 3, 4, 8, 9], inf)));

n.sendMsg("/oo", \Pdef_x, \play);

Ndef(\zz, { |freq=440, numharm = 200| Blip.ar(freq * [1, 1.2], numharm) * 0.3 } );

n.sendMsg("/oo", \Ndef_zz, \play);

n.sendMsg("/oo", \Ndef_zz, \fadeTime_, 5);

n.sendMsg("/oo", \Ndef_zz, \xset, \freq, 174);

n.sendMsg("/oo", \Ndef_zz, \end);



Interpreting


For remote code execution, OpenObject can be opened to the interpreter. 

    Use with care! 

    (any code can be executed from the network).


openInterpreterFor(addr, cmd, selector)

addr: address from which the messages come

cmd: OSC command name

selector: selector which is called on the object for the interpreted string

cmd name string



// example

OpenObject.openInterpreterFor(nil, '/oosrc', 'source_');


Ndef(\y, { |freq=440, numharm = 200| Blip.ar(freq * [1, 1.2], numharm) * 0.3 } );

Ndef(\y).publish(\brussels);

n.sendMsg("/oo", \brussels, \play);


// set the source:

n.sendMsg("/oosrc", \brussels, "{ |freq = 1000| Ringz.ar(Dust.ar(10.dup), freq, 0.07) }");


// this is equivalent to:

Ndef(\y).source_({ |freq = 1000| Ringz.ar(Dust.ar(10.dup), freq, 0.07) });
