VoicerProxy


A place-holder for a [Voicer]. Created automatically when a Voicer is .gui'ed.


The structure of Voicer GUI's and MIDI objects is quite complex, but transparent to use. It's helpful to understand it to know how to control the remapping of MIDI controllers, sequences, and GUI objects.


Voicer ------- VoicerProxy ------- VoicerProxyGui


A VoicerGUI cannot exist without a VoicerProxy.


VoicerProxyGui ------- [VoicerProcessGroup]s


VoicerProcessGroups work only through a proxy. This is so the same sequencer can play through any Voicer.


VoicerProxyGui ------- [VoicerGCProxy]s ------- [VoicerGlobalControl]s


The GUI knows only about proxies for the GlobalControls, so that the gui objects can be reassigned easily.


VoicerMIDISocket ------- Voicer

------- VoicerProxy


MIDI sockets can cling to one Voicer for dear life, or can be remapped through the proxy.


VoicerMIDIController ------- VoicerGlobalControl

------- VoicerGCProxy


Creation:


A VoicerProxy is created by:


aVoicer.gui

aVoicer.addProcess()

aVoicer.proxify


After these operations, the proxy can be accessed by aVoicer.proxy, or:


aProxy = aVoicer.proxify.


Proxify returns the existing proxy if it exists, or creates a new one if needed.


Assignment:


aProxy.voicer = aVoicer; // everything updates: gui, MIDI, controls, etc.


Freeing:


aProxy.free; // assigns the Voicer's proxy to nil (and global controls), frees processes, and removes gui
