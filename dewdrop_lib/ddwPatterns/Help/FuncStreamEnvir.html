FuncStreamEnvir : FuncStream


A FuncStream is created when a stream is created from a Pfunc. This can be a problem if the Pfunc depends on values in a specific environment. FuncStream does not remember the active environment when it is created; thus future calls to .next may not retrieve the correct values because the active environment will be different at .next time.


FuncStreamEnvir does remember the proper environment. This is especially helpful when using patterns inside AdhocClass objects.


dewdrop_lib overrides Pfunc's asStream method to use FuncStreamEnvir instead of FuncStream.


Example:


// Using FuncStream, math operation fails

(

var envir = (

a: 1,

makeStream: { FuncStream({ ~a * 5 }) }

),

stream = envir.use({ ~makeStream.value });


stream.next;

)


// Using FuncStreamEnvir, the ~a environment variable is available

(

var envir = (

a: 1,

makeStream: { FuncStreamEnvir({ ~a * 5 }) }

),

stream = envir.use({ ~makeStream.value });


stream.next;

)


// Pfunc uses FuncStreamEnvir when dewdrop_lib is installed

(

var envir = (

a: 1,

makePattern: { Pfunc({ ~a * 5 }) }

),

stream = envir.use({ ~makePattern.value.asStream });


stream.next;

)
