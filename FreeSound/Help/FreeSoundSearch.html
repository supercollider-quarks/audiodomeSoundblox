FreeSoundSearch GUI front-end for FreeSound class


Inherits from: Object


This is a simple front-end for the FreeSound class.


See also: FreeSound


Creation / Class Methods


*new (argDoneFunc)

It gets a function as an argument. This function is passed in two arguments, when the search and download tasks are finished. The first argument is the parent FreeSoundSearch instance, and the second argument is an array of paths pointing to the downloaded files.

g = FreeSoundSearch({|fsInstance, paths| paths.postln; });

//the only instance methods:

g.hideGui;

g.showGui;


Some Notes:

The keywords are separated by commas. For every keyword, this front-end tries to download one sample, so if you give 5 keywords, it will download 5 samples.

After the results are returned, their properties are evaluated in an order defined by the bottom evaluation order menu. The specs of the found files are compared with the dictionary inside "Acceptable File Specs" text view. To see what the keys mean, see the help for FreeSound. For the front-end to decide to download a sample, the properties of that sample should match with the ones user provides in this provided dictionary. The items can have functions to evaluate a spec, or you can use \any that matches all. Your functions should return "true" if you accept the passed in argument. All \any values return true by default, and all the other items inside the dictionary should return "true" for the front-end to download a particular file.
